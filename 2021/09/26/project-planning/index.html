Project Planning for a single developer

26 Sep 2021

Problem

I am trying to find a better way to structure the way I work as a single developer. I find that I have a lot of “stuck thinking” time because I wear many hats at the same time. When I should be doing programming, I’ll sometimes be stuck thinking about game design or about the graphics. Or I’ll be stuck thinking “should I be doing something else?!”.

The good thing about working as a team is that people decide as a team and then you go do your work. If you think you should be working on something else, then you need to wait to talk about that in the next meeting. But if you’re a single developer you can be very agile and change what you are doing freely. This creates a lot of flexibility but it also makes a lot of things bad.

Bad performance because of context-switching

thinking about project management or game design while I should be programming

Wandering in the desert

Allowing me to change what I am doing in the heat of the moment or get stuck thinking about it makes it more likely I am not working towards the “vision of the project”

Hard to commit

I will sometimes schedule time for marketing but then make a decision that “I don’t need to do marketing lately because of X”, then go on to program. I think that by delaying that decision to later, I can make a more informed decision.

Solution

Weekly public management blog

Write weekly about how the week went and post times for each profession.

Schedule time for each profession

Every profession should try to do only their minimal time, while the programmer can work as much as they want. If there is not enough time to do the work, they should write a note for the team meeting.

Game Designer

  • Decides what will be put in the game
  • Play tests, brainstorms, rebalances

Programmer

  • Programs functionality decided by the game designer

Automation Programmer

  • Programs functionality decided by the manager

Marketer

  • Writes blog posts
  • Takes screenshots, gifs and videos
  • Uploads builds
Manager - Team Meeting
  • Checks if every job is doing their weekly time
  • Rethinks how time should be spent in the project
  • Decides what the automation programmer should make
  • Assigns tasks to everyone
  • Write the weekly write up
  • Rethinks this whole team planning thing and considers if reverting to the flexible style is better
Producer
  • Checks if the game is going according to vision

Conclusion

Let us see how well this works hahahaha good luck, me.
